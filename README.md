# Road Sign Recognition (PyTorch CNN + YOLOv8)

End-to-end road sign recognition system with two models:

1. **Detector (YOLOv8):** finds sign locations in a full image.
2. **Classifier (PyTorch CNN):** predicts sign class from each detected crop.

The project is built for your current GTSRB-style dataset layout and runs on Windows with GPU-enabled PyTorch.

---

## What This Project Does

- Prepares your dataset into:
	- Classification format (`prepared/classification/train|val|test/<class>/...`)
	- YOLO format (`dataset_yolo/images/...` + `dataset_yolo/labels/...`)
- Trains a custom CNN classifier that matches the architecture specified in your prompt.
- Trains a YOLOv8 detector for bounding box localization.
- Combines both models into one inference pipeline.
- Provides evaluation reports, Grad-CAM visualization, real-time webcam inference, and a Streamlit app.

## Dataset Assumptions

This repository expects the dataset in this format:

```text
dataset/
	Train/
		0/ ...png
		1/ ...png
		...
		42/ ...png
	Test/
		...png
	Meta/
		...png
	Train.csv
	Test.csv
	Meta.csv
```

### Important notes

- `Train.csv` and `Test.csv` are read and merged by `src.preprocess`.
- A new stratified split is created: **70% train / 15% val / 15% test**.
- YOLO labels are generated from ROI columns in CSV (`Roi.X1`, `Roi.Y1`, `Roi.X2`, `Roi.Y2`).
- Running preprocessing will recreate output folders under `prepared/` and `dataset_yolo/`.

## Model Architecture (Classifier)

The custom PyTorch CNN follows the prompt architecture:

- **Input:** `(64, 64, 3)`
- **Block 1:**
	- Conv2d(3->32, k=3, padding=1), ReLU
	- BatchNorm2d(32)
	- Conv2d(32->32, k=3, padding=1), ReLU
	- MaxPool2d(2)
	- Dropout(0.25)
- **Block 2:**
	- Conv2d(32->64, k=3, padding=1), ReLU
	- BatchNorm2d(64)
	- Conv2d(64->64, k=3, padding=1), ReLU
	- MaxPool2d(2)
	- Dropout(0.25)
- **Block 3:**
	- Conv2d(64->128, k=3, padding=1), ReLU
	- BatchNorm2d(128)
	- MaxPool2d(2)
	- Dropout(0.40)
- **Head:**
	- Flatten
	- Linear -> 512, ReLU
	- BatchNorm1d(512)
	- Dropout(0.50)
	- Linear -> `num_classes`

Loss and optimization:

- Loss: `CrossEntropyLoss`
- Optimizer: `Adam(lr=0.001)`
- LR scheduler: `ReduceLROnPlateau(factor=0.5, patience=5, min_lr=1e-6)`
- Early stopping: patience 10 (val loss)

Transfer learning option:

- `--transfer-learning` uses MobileNetV2 backbone.
- Base is frozen first, then last 30 leaf layers are fine-tuned with lower LR.

## Full Workflow (Phase by Phase)

Run commands from project root.

### Phase 1 + 2: Prepare data

```powershell
.\.venv\Scripts\python.exe -m src.preprocess
```

Useful optional args:

```powershell
.\.venv\Scripts\python.exe -m src.preprocess --train-ratio 0.7 --val-ratio 0.15 --test-ratio 0.15
.\.venv\Scripts\python.exe -m src.preprocess --no-hardlinks
```

Outputs:

- `prepared/classification/train|val|test/...`
- `dataset_yolo/images/...`
- `dataset_yolo/labels/...`
- `artifacts/class_names.json`
- `artifacts/class_id_map.json`
- `artifacts/splits/train.csv`, `val.csv`, `test.csv`
- `data.yaml`

### Phase 3: Train classifier (custom CNN)

```powershell
.\.venv\Scripts\python.exe -m src.train_cnn
```

Common options:

```powershell
.\.venv\Scripts\python.exe -m src.train_cnn --epochs 50 --batch-size 32 --image-size 64
.\.venv\Scripts\python.exe -m src.train_cnn --device cuda
```

Transfer learning option:

```powershell
.\.venv\Scripts\python.exe -m src.train_cnn --transfer-learning --fine-tune-epochs 10
```

Outputs:

- `models/best_cnn_model.pth`
- `models/final_cnn_model.pth`
- `reports/cnn_history.csv`
- `reports/cnn_training_curves.png`
- `artifacts/cnn_label_map.json`

### Phase 4: Train YOLO detector

```powershell
.\.venv\Scripts\python.exe -m src.train_yolo 
.\.venv\Scripts\python.exe -m src.train_yolo --name road_sign_v2_full --epochs 100 --fraction 1.0 --patience 20 --batch 32 --workers 2 --device 0 --cache disk --optimizer Adam --lr0 0.01 --lrf 0.001 --exist-ok
```

Default settings are tuned for a 4GB RTX 3050:

- model: `yolov8n.pt`
- `imgsz=416`
- `batch=32`
- `workers=2`
- `device=0` (GPU)
- `cache=disk`
- `patience=20`
- `optimizer=Adam`
- `fraction=1.0`
- `exist_ok=True`

Useful options:

```powershell
.\.venv\Scripts\python.exe -m src.train_yolo --name road_sign_v1_quick --epochs 50 --fraction 0.5 --patience 15
.\.venv\Scripts\python.exe -m src.train_yolo --cache ram
.\.venv\Scripts\python.exe -m src.train_yolo --cache none
.\.venv\Scripts\python.exe -m src.train_yolo --cpu-fallback
```

Recommended run plan:


## Training and Runtime Tips

- If CUDA is available, scripts auto-select GPU unless overridden.
- If you get out-of-memory errors, reduce:
	- CNN: `--batch-size`
	- YOLO: `--batch`, optionally `--imgsz`
- For faster data loading, use `--num-workers` > 0 in `train_cnn` and `evaluate`.
- Keep `image-size=64` for architecture consistency with project prompt.

## Troubleshooting

### PyTorch installs CPU build instead of GPU

- Ensure `requirements.txt` still contains:
	- `--index-url https://download.pytorch.org/whl/cu124`
	- `torch==2.6.0+cu124`
	- `torchvision==0.21.0+cu124`
- Reinstall with:

```powershell
.\.venv\Scripts\python.exe -m pip install --upgrade --force-reinstall -r requirements.txt
```

### `ModuleNotFoundError`

- Make sure you use the venv Python in commands (`.\.venv\Scripts\python.exe ...`).

### Missing `data.yaml` or prepared data

- Run preprocessing first:

```powershell
.\.venv\Scripts\python.exe -m src.preprocess
```

### Wrong class labels at inference

- Ensure preprocessing and training were run from the same dataset state.
- Verify `artifacts/class_names.json` and `artifacts/cnn_label_map.json` exist.
